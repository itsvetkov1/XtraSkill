## Overview

Implemented deep linking and URL preservation for the BA Assistant. Users can bookmark, share, and refresh URLs without losing their place. The OAuth authentication flow preserves intended destinations, and invalid routes show helpful error states.

---

## Accomplishments

1. **Deep Linking Routes** - Conversations accessible via unique URLs (`/projects/:projectId/threads/:threadId`)
2. **404 Error Page** - GoRouter errorBuilder with custom NotFoundScreen showing attempted path
3. **URL Preservation Service** - UrlStorageService using sessionStorage to preserve URLs through OAuth
4. **Return URL Flow** - Login captures intended destination and redirects after auth completes
5. **Resource Not Found States** - Graceful handling of deleted projects/threads with ResourceNotFoundState widget
6. **Production Deployment Guide** - Server configuration documentation for nginx, Apache, Vercel, Firebase

### Phase 16: auth-url-preservation

**Goal:** Users can refresh pages or return from OAuth login without losing their intended destination.

**Requirements:**
- URL-01: Page refresh preserves current URL for authenticated users
- URL-02: URL preserved through OAuth redirect via session storage
- URL-03: Settings page refresh returns to `/settings` (not `/home`)
- URL-04: Project detail refresh returns to `/projects/:id` (not `/home`)
- AUTH-01: Auth redirect captures intended destination as `returnUrl` query parameter
- AUTH-02: Login completes to stored `returnUrl` instead of `/home`
- AUTH-03: Direct login (no returnUrl) goes to `/home`
- AUTH-04: `returnUrl` cleared from session storage after successful navigation

**Plans:**
- 16-01-PLAN.md - UrlStorageService and redirect logic foundation
- 16-02-PLAN.md - Login/Callback screen returnUrl integration

**Key Files Created:**
